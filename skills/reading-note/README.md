# reading-note

本、PDF、記事、文書を「精読未満だが、精読の代替として使える水準」で理解するためのノートを作る skill です。

## 使う場面

- 本やPDFの精読代替ノートを作りたい。
- テーマ、論理の流れ、章ごとの役割を整理したい。
- 登場概念を、定義だけでなく「なぜ必要か」まで理解したい。
- 「XXをYYできる」という理解ゴールに分解したい。
- 初見では圧縮要約が頭に入らないため、導入から順に理解したい。

## 出力の基本形

1. この本は何を作る本か
2. 全体構造
3. ひとつの単純な例
4. 問いの連鎖
5. 圧縮版の論理の流れ
6. 章ごとの役割
7. 重要概念台帳
8. 理論と実務の接続
9. 最小公式・手順集
10. 理解ゴール一覧
11. 検証メモ
12. 自己テスト
13. 1分説明

## 重要な考え方

この skill は、圧縮された論理の流れから始めません。初見者が理解できるように、まず「この本は何を作る本か」「全体構造」「単純な例」「問いの連鎖」を置き、その後に圧縮ロジックを提示します。

用語も単なる一覧にはしません。各概念について、平たく言うと何か、なぜ必要か、論理上どこに置かれるか、誤解すると何が壊れるか、何ができるようになるかを整理します。

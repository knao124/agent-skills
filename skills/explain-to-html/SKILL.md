# Explain To Html

## Overview

Use this skill when the explanation itself should be persisted as a browser-viewable file. Investigate the target first, then write a concise standalone HTML page under `/tmp` and reply with the absolute file path plus a short summary in chat.

The generated page is also an interactive review surface. Readers must be able to add Figma-like comments to individual explanation components, then export those comments as a prompt they can paste into an AI assistant to request improvements.

## Output Contract

- Always create a UTF-8 HTML file under `/tmp`.
- Prefer the filename pattern `/tmp/codex-explain-<topic>-<YYYYMMDD-HHMMSS>.html`.
- Keep the page standalone: inline CSS and inline JavaScript only, no external assets, no network fetches, no frameworks.
- Default to a concise engineer-facing explanation unless the user specifies another audience.
- Every generated HTML file must include the interactive review layer described below. If the user explicitly forbids JavaScript, ask before proceeding because that conflicts with this skill's required review behavior.
- If the user explicitly asks for plain text only or says not to write a file, do not use this skill.
- Reply in chat with:
  1. the absolute file path
  2. one short summary of what the file contains
  3. any important caveat that did not fit naturally in the HTML

## Workflow

1. Inspect the requested target enough to explain it accurately.
2. Decide the scope:
   - default: explain what it does, how it works, and the important moving parts
   - for code: include relevant file paths and line references when they materially help
   - for commands or configs: explain purpose, inputs, outputs, and operational caveats
3. Identify review components before writing:
   - mark each major section, code block group, diagram, example, risk note, or recommendation as a component
   - add stable, unique `data-review-target` IDs such as `overview`, `flow-auth-check`, or `risk-cache-invalidation`
   - add human-readable `data-review-label` values that can appear in exported prompts
4. Draft a short structure before writing:
   - title
   - summary
   - main sections
   - optional risks, tradeoffs, or next steps
5. Write the HTML file to `/tmp`.
6. Verify the file exists after writing it and that it contains `data-review-target`, `review-toggle`, and `review-export`.
7. Return the path in chat with a 1-3 line summary instead of pasting the full explanation into the conversation.

## Required Interactive Review Layer

Every HTML file generated by this skill must include these features:

- Component-level targets: each meaningful `header`, `section`, `article`, `figure`, or grouped `pre` block has a unique `data-review-target="<stable-id>"` and `data-review-label="<readable label>"`.
- Review mode toggle: a fixed toolbar button turns component outlines and comment pins on or off.
- Figma-like comments: each component gets an interactive pin button. Clicking it opens a side panel for that component, shows existing comments, and lets the reader add or remove comments.
- Local persistence: comments are stored in `localStorage` using a key derived from `location.pathname` and `document.title`, so the standalone HTML file needs no backend.
- Improvement prompt export: a toolbar button opens a copyable prompt containing the collection of `{target label, target id, target excerpt, comment}` pairs. The prompt must be useful when pasted into an AI assistant.
- Clipboard fallback: provide a visible textarea with the full prompt even if `navigator.clipboard.writeText` is unavailable.

Use short labels in the review UI, such as `コメント`, `改善プロンプト`, `コピー`, and `閉じる`. Do not add long usage instructions to the page.

## Recommended Page Structure

- `header`
  - title
  - generated timestamp
  - target summary
- `main`
  - `Overview`
  - `How It Works`
  - `Important Details`
  - `Examples` when concrete examples improve comprehension
  - `Risks / Notes` when there are caveats
- `footer`
  - referenced files, commands, or links when useful
- fixed review toolbar
  - `コメント` toggle
  - `改善プロンプト` export button
- review side panel and export dialog near the end of `body`

Prefer short sections and readable spacing over decorative styling.

## HTML Template

Use a semantic structure like this and adapt the contents to the task. Keep the review layer intact.

```html

    <script>
      (() => {
        const storageKey = `codex-explain-comments:${location.pathname}:${document.title}`;
        const targets = [...document.querySelectorAll("[data-review-target]")];
        const byId = new Map(targets.map((el) => [el.dataset.reviewTarget, el]));
        const $ = (selector) => document.querySelector(selector);
        const escapeHtml = (value) =>
          String(value).replace(/[&<>"']/g, (ch) => ({ "&": "&amp;", "<": "&lt;", ">": "&gt;", '"': "&quot;", "'": "&#39;" }[ch]));
        const load = () => {
          try {
            return JSON.parse(localStorage.getItem(storageKey) || "[]");
          } catch {
            return [];
          }
        };
        const state = { comments: load(), activeTarget: null };
        const save = () => {
          try {
            localStorage.setItem(storageKey, JSON.stringify(state.comments));
          } catch {
            // Keep the review UI usable even when localStorage is blocked.
          }
        };
        const labelFor = (el) => el?.dataset.reviewLabel || el?.querySelector("h1,h2,h3")?.textContent.trim() || el?.dataset.reviewTarget || "Target";
        const excerptFor = (el) => (el?.innerText || "").replace(/\s+/g, " ").trim().slice(0, 280);

        targets.forEach((el) => {
          const pin = document.createElement("button");
          pin.type = "button";
          pin.className = "review-pin";
          pin.dataset.reviewOpen = el.dataset.reviewTarget;
          pin.setAttribute("aria-label", `${labelFor(el)} にコメント`);
          pin.textContent = "+";
          el.appendChild(pin);
        });

        const renderPins = () => {
          targets.forEach((el) => {
            const count = state.comments.filter((comment) => comment.target === el.dataset.reviewTarget).length;
            const pin = el.querySelector(".review-pin");
            pin.classList.toggle("has-comments", count > 0);
            pin.textContent = count > 0 ? String(count) : "+";
          });
        };

        const renderPanel = () => {
          const targetEl = byId.get(state.activeTarget);
          $("#review-panel-title").textContent = labelFor(targetEl);
          $("#review-panel-target").textContent = state.activeTarget || "";
          const comments = state.comments.filter((comment) => comment.target === state.activeTarget);
          $("#review-comment-list").innerHTML = comments
            .map((comment) => `<div class="review-comment"><p>${escapeHtml(comment.text)}</p><button type="button" data-delete-comment="${comment.id}">削除</button></div>`)
            .join("");
        };

        const openPanel = (target) => {
          state.activeTarget = target;
          $("#review-panel").hidden = false;
          renderPanel();
          $("#review-text").focus();
        };

        document.body.addEventListener("click", (event) => {
          const openButton = event.target.closest("[data-review-open]");
          if (openButton) openPanel(openButton.dataset.reviewOpen);
          const deleteButton = event.target.closest("[data-delete-comment]");
          if (deleteButton) {
            state.comments = state.comments.filter((comment) => comment.id !== deleteButton.dataset.deleteComment);
            save();
            renderPanel();
            renderPins();
          }
        });

        $("#review-toggle").addEventListener("click", () => {
          document.body.classList.toggle("review-mode");
        });
        $("#review-close").addEventListener("click", () => {
          $("#review-panel").hidden = true;
        });
        $("#review-form").addEventListener("submit", (event) => {
          event.preventDefault();
          const text = $("#review-text").value.trim();
          if (!text || !state.activeTarget) return;
          state.comments.push({ id: `${Date.now()}-${Math.random().toString(16).slice(2)}`, target: state.activeTarget, text });
          $("#review-text").value = "";
          save();
          renderPanel();
          renderPins();
        });

        const buildPrompt = () => {
          if (state.comments.length === 0) return "このHTML解説には、まだ改善コメントがありません。";
          const blocks = state.comments.map((comment, index) => {
            const targetEl = byId.get(comment.target);
            return [
              `${index + 1}. 対象: ${labelFor(targetEl)} (${comment.target})`,
              `該当箇所: ${excerptFor(targetEl)}`,
              `コメント: ${comment.text}`,
            ].join("\n");
          });
          return ["以下のHTML解説を改善してください。各項目は、指摘対象とコメント内容のセットです。", ...blocks].join("\n\n");
        };

        $("#review-export").addEventListener("click", () => {
          $("#review-export-text").value = buildPrompt();
          $("#review-export-panel").hidden = false;
          $("#review-export-text").focus();
          $("#review-export-text").select();
        });
        $("#review-export-close").addEventListener("click", () => {
          $("#review-export-panel").hidden = true;
        });
        $("#review-copy").addEventListener("click", async () => {
          $("#review-export-text").select();
          try {
            await navigator.clipboard.writeText($("#review-export-text").value);
          } catch {
            document.execCommand("copy");
          }
        });

        renderPins();
      })();
    </script>
  </body>
</html>
```

## Writing Rules

- Make the page readable on both desktop and mobile.
- Keep CSS inline and moderate in size.
- Keep JavaScript inline, small, and limited to the review/comment/export behavior.
- Prefer neutral, durable explanations over rhetorical prose.
- When citing local files, use absolute paths and line numbers in the HTML body when available.
- Escape or fence code snippets correctly.
- Avoid including hidden reasoning, secrets, tokens, or irrelevant tool output dumps.
- Do not omit the component comment UI or improvement prompt export for brevity.

## Example Triggers

- `この関数を解説して`
- `この diff を解説して。HTML で見たい`
- `この設定ファイルを説明して`
- `このコマンドが何をしているか解説して`
- `この設計をHTMLで解説して、コメントから改善プロンプトも出せるようにして`
